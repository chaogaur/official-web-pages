# official-web-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可以用于展示、测试、文档引用或其他通用用途。仓库本身不绑定任何特定域名或网站，仅作为页面文件的集中存放地。

## 项目简介

`official-web-pages` 是一个轻量级的页面归档项目。仓库内包含若干 HTML 文件，每个文件均可独立运行或通过浏览器直接打开。适合存放静态页面、临时演示页面、技术测试页面等。

## 目录结构

```
official-web-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 可选：页面所使用的资源文件（如 CSS、图片等）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立的页面，文件名通常反映页面用途或内容。
- `assets/`：如果页面需要共享资源（如样式表或图片），可统一放置于此目录，便于维护。

## 页面归档说明

- 所有页面均为纯 HTML 文件，不依赖服务器端处理。
- 页面之间相互独立，无跨页面依赖。
- 文件命名应简洁清晰，避免特殊字符。
- 如需引用外部资源（如字体、库文件），建议使用相对路径或保持资源本地化。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保文件可独立运行。
- 更新页面：直接修改对应 HTML 文件，注意保持文件结构和命名规范。
- 删除页面：移除不需要的 HTML 文件，同时清理 `assets/` 中不再使用的资源。
- 版本控制：所有变更通过 Git 提交记录，建议在提交信息中简要说明页面内容变化。

## 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中直接打开 `pages/` 目录下的任意 HTML 文件即可查看。
3. 也可将仓库部署至静态托管服务，通过 URL 访问对应页面。

## 注意事项

- 本仓库不包含任何动态后端逻辑，所有页面均为前端静态内容。
- 请勿在页面中加入恶意代码或违反法律法规的内容。
- 资源文件请尽量保持轻量，避免仓库体积过大。
